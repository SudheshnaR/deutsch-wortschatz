## 1. Scope

This report records automated testing of the Deutsch Wortschatz application — a German vocabulary learning app delivered as a Capacitor web app that ships to **web, Android, and iOS from a single codebase** (`www/index.html`). It covers the three content levels (A1, A2, B1), the themed phrasebook, the spaced-repetition engine, onboarding and profiles (including the vocabulary filter and PIN lock), data persistence and backup, the daily story feature, and build/config integrity.

The suite is **executable**: it loads the application's *real* JavaScript and data into a mocked-browser sandbox (Node.js) and asserts actual behaviour. Every result below was produced by running the code — not by inspection.

## 2. Method

`tests/run.js` extracts the inline `<script>` from `www/index.html`, evaluates it inside a Node `vm` sandbox with lightweight mocks for the browser environment (`document`, `localStorage`, `speechSynthesis`, timers, Capacitor), then exercises the exposed data and pure logic — word lists, the SRS engine, the vocabulary filter, sessions, the data store, profiles, custom words, and the story rotation. Date-dependent behaviour is driven through the app's own `simOffset` test hook. Each test has an ID, an area, a description, and a live PASS/FAIL result.

**Run it yourself:**
```bash
npm test          # runs the suite, prints the table, exits non-zero on any failure
npm run test:report   # runs the suite and regenerates this document
```
